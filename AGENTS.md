# Архитектура проекта: Polzuchie-tvari.io (Snake AI)

Проект представляет собой мультиплеерную игру "Змейка" в реальном времени.

## Структура проекта

Проект разделен на две основные части: `backend` и `frontend`.

### Backend (Серверная часть)
Расположение: `/backend`
Стек: Python, FastAPI, Uvicorn, WebSockets.

- **`backend/server.py`**: Точка входа и основная логика сервера. 
  - Реализует класс `GameState`, который управляет игровым полем (размер 100x100), спавном еды, физикой движения и столкновениями, используя оптимизацию **Пространственного разделения (Spatial Partitioning)** с размером ячейки сетки 10.0.
  - Игровой цикл (`game_loop`) работает асинхронно с частотой **20 тиков в секунду (0.05s tick rate)**. 
  - Для оптимизации сети внедрены **Area of Interest (AoI)** (клиент получает только то, что рядом) и **Настоящая Дельта (True Delta)** (вместо всего хвоста сервер шлет только координаты новых добавленных голов).
  - Управление событиями клиентов (подключение, отключение, смена направления) происходит в эндпоинте `/ws/{client_id}`.
  - Горячая перезагрузка (Hot-reload) сервера управляется переменной окружения `ENVIRONMENT` (отключается в продакшене).
- **`backend/requirements.txt`**: Зависимости (`fastapi`, `uvicorn[standard]`, `msgpack`, `websockets`).

### Frontend (Клиентская часть)
Расположение: `/frontend`
Стек: Next.js (App Router), React, PixiJS (WebGL), HTML5 Canvas (для радара).

- **`frontend/app/page.tsx`**: Основной интерфейс игры и рендеринг.
  - Устанавливает WebSocket-соединение с сервером.
  - Обрабатывает ввод пользователя: стрелки или A/D для поворота, клавиша "C" для переключения режимов камеры (2D/3D).
  - **Предсказание и Интерполяция:** Применяется **Client-Side Prediction (CSP)** — клиент локально предсказывает угол поворота своей змейки для мгновенного отклика (0 мс задержки) и мягко интерполирует движения остальных змей между серверными тиками.
  - **Рендеринг (WebGL/PixiJS):** Основная отрисовка происходит через аппаратное ускорение GPU с помощью PixiJS. Для эффекта 3D применяется CSS-перспектива `transform: perspective`, а туман отрисовывается в отдельном аппаратном CSS-слое поверх сцены.
  - **Радар и Лидерборд:** Радар рисуется на классическом 2D Canvas. Для змей, находящихся вне поля зрения, сервер присылает урезанные массивы тела (только голова 1 сегмент), чтобы сэкономить трафик, но сохранить их отображение на радаре.
- **`frontend/package.json`**: Зависимости фронтенда (React 19, Next.js 16).

### Инфраструктура и Деплой (Docker & Nginx)
- **`docker-compose.yml` & `.env`**: Оркестрация контейнеров `backend` и `frontend`. Внешние порты привязываются строго к `127.0.0.1` (доступ извне идет только через Nginx).
- **Dockerfiles**: Сборка изолированных образов. Фронтенд собирается в максимально легковесном режиме `standalone` (указано в `next.config.mjs`).
- **Nginx (Reverse Proxy)**: В продакшене используется для SSL (HTTPS) и маршрутизации. Запросы на `/` идут на фронт (порт 3000), а пути `/ws/` проксируются на бэкенд (порт 8000).
- **Динамические WebSockets**: Клиент (`useGameSocket.ts`) сам определяет среду по порту браузера (стандартные 80/443 -> прокси через Nginx по `wss://`, иначе напрямую на локальный `:8000`).

## Ключевые механизмы и коммуникация
1. **Сетевой протокол:** WebSockets.
2. **Формат данных:** Используется бинарный формат **MessagePack**. Сервер отправляет тип состояния `"FULL"` при входе и тип `"DELTA"` при обновлениях.
3. **Механика гибели:** При столкновении змейка оставляет 50% от своих набранных очков в виде яблок (номиналом 1, 2, 5, 10, 20, 50 очков), после чего моментально возрождается со сброшенными параметрами.
4. **Рост:** Змейка увеличивается на 1 сегмент за каждые 10 съеденных очков. Толщина змейки и радиус ее головы динамически увеличиваются в зависимости от счета.
5. **Ускорение:** При зажатом пробеле (или Space) змейка движется в 2 раза быстрее, но каждую секунду сбрасывает часть своей массы в виде еды позади хвоста.
